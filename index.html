üåê Pengenalan React & Vite

Selamat datang di dunia React! Di bagian ini, kita akan menjelajahi apa itu React, mengapa ia menjadi pilihan populer untuk membangun antarmuka pengguna, dan bagaimana Anda bisa dengan cepat memulai proyek React menggunakan Vite.

Apa itu React? ‚öõÔ∏è

React adalah sebuah pustaka (library) JavaScript yang deklaratif, efisien, dan fleksibel yang dikembangkan oleh Facebook. Tujuan utamanya adalah untuk membangun antarmuka pengguna (User Interfaces - UI) yang kompleks dan interaktif dari potongan-potongan kode kecil yang disebut komponen.

Pendekatan deklaratif React memungkinkan Anda untuk mendeskripsikan bagaimana UI seharusnya terlihat berdasarkan data (state) aplikasi saat ini. React kemudian akan secara cerdas dan efisien memperbarui DOM (Document Object Model) hanya ketika data tersebut berubah, memastikan aplikasi Anda tetap cepat dan responsif.

Mengapa React Begitu Populer?

  • Virtual DOM: React menggunakan konsep Virtual DOM untuk optimasi performa. Alih-alih memanipulasi DOM browser secara langsung (yang bisa lambat), React membuat representasi DOM di memori. Ketika state berubah, React memperbarui Virtual DOM, membandingkannya dengan versi sebelumnya (proses "diffing"), dan kemudian hanya menerapkan perubahan yang diperlukan ke DOM browser yang sebenarnya (proses "reconciliation").
  • Arsitektur Berbasis Komponen: Memecah UI menjadi komponen-komponen kecil yang independen, dapat digunakan kembali (reusable), dan memiliki logikanya sendiri. Ini membuat kode lebih modular, mudah dikelola, diuji, dan diskalakan seiring pertumbuhan aplikasi.
  • Ekosistem yang Luas dan Aktif: React memiliki komunitas developer yang sangat besar dan aktif. Ini berarti banyak sekali sumber daya pembelajaran, tutorial, pustaka pihak ketiga (seperti React Router untuk navigasi, Redux/Zustand untuk manajemen state yang kompleks), dan dukungan yang tersedia secara online.
  • Unidirectional Data Flow (Aliran Data Satu Arah): Data di aplikasi React umumnya mengalir dalam satu arah (biasanya dari komponen induk ke komponen anak melalui props). Ini membuat state aplikasi lebih mudah diprediksi, dipahami, dan di-debug.
  • Didukung oleh Facebook dan Komunitas: Pengembangan dan pemeliharaan React yang berkelanjutan oleh Facebook dan kontribusi dari komunitas global memastikan relevansi dan kemajuan teknologi ini.

Memulai Proyek React dengan Vite ‚ö°

Vite adalah alat bantu pengembangan web modern yang menawarkan pengalaman pengembangan frontend yang sangat cepat. Ia menyediakan server pengembangan dengan Hot Module Replacement (HMR) yang hampir instan dan proses build yang sangat dioptimalkan.

Untuk membuat proyek React baru menggunakan Vite, buka terminal Anda dan jalankan perintah berikut:

# Ganti 'nama-proyek-anda' dengan nama direktori yang diinginkan
npm create vite@latest nama-proyek-anda -- --template react

Setelah proyek berhasil dibuat, navigasikan ke direktori proyek tersebut:

cd nama-proyek-anda

Kemudian, instal semua dependensi yang dibutuhkan oleh proyek:

npm install

Terakhir, jalankan server pengembangan Vite:

npm run dev

Vite akan memulai server pengembangan, dan aplikasi React Anda akan dapat diakses melalui browser, biasanya pada alamat http://localhost:5173. Anda akan melihat halaman selamat datang React yang siap untuk Anda modifikasi.

‚ú® JSX: JavaScript XML

JSX adalah ekstensi sintaks untuk JavaScript yang sangat direkomendasikan (dan hampir selalu digunakan) saat bekerja dengan React. Ini memungkinkan Anda menulis markup mirip HTML langsung di dalam file JavaScript Anda, membuat kode untuk UI lebih intuitif dan mudah dibaca.

Memahami JSX Lebih Dalam

Meskipun terlihat seperti HTML, JSX bukanlah HTML. Browser tidak dapat memahami JSX secara langsung. JSX adalah "syntactic sugar" (gula sintaksis) untuk fungsi React.createElement(component, props, ...children). Sebelum kode dijalankan di browser, JSX perlu dikompilasi (ditranspilasi) menjadi panggilan fungsi JavaScript biasa oleh alat seperti Babel (yang sudah terintegrasi dalam proyek Vite).

Contoh JSX dan Hasil Kompilasinya:

const nama = "Pengguna React";
const elemen = <h1 className="salam">Halo, {nama}!</h1>;

// JSX di atas, setelah dikompilasi, akan menjadi kurang lebih seperti ini:
// React.createElement("h1", {className: "salam"}, "Halo, ", nama, "!");

Menggunakan JSX membuat struktur komponen lebih visual dan mirip dengan hasil akhir HTML yang akan dirender, yang sangat membantu dalam pengembangan UI.

Aturan Penting dalam Penggunaan JSX

  • Harus Ada Satu Elemen Induk (Root Element): Setiap komponen React yang me-render JSX harus mengembalikan satu elemen induk tunggal. Jika Anda perlu mengembalikan beberapa elemen sejajar, Anda harus membungkusnya dalam satu elemen pembungkus (misalnya, <div>) atau menggunakan <React.Fragment> (atau sintaks singkatnya <>...</>) yang tidak menambahkan node ekstra ke DOM. 
    // Benar
function MyComponent() {
  return (
    <>
      <h1>Judul</h1>
      <p>Paragraf.</p>
    </>
  );
}
  • Atribut HTML Ditulis dalam camelCase: Banyak atribut HTML standar berfungsi di JSX, tetapi beberapa yang terdiri dari dua kata atau lebih ditulis dalam format camelCase. Contohnya, atribut class di HTML menjadi className di JSX (karena class adalah kata kunci di JavaScript), dan for (untuk label) menjadi htmlFor. Atribut seperti onclick menjadi onClick.
  • Menyematkan Ekspresi JavaScript dengan `{}`: Anda dapat menyematkan ekspresi JavaScript apa pun (variabel, hasil fungsi, operasi matematika, kondisi ternary) di dalam JSX dengan membungkusnya dalam kurung kurawal { }. 
    const linkGambar = "path/ke/gambar.jpg";
const sudahLogin = true;
const elemen = (
  <div>
    <img src={linkGambar} alt="Deskripsi" />
    <p>{sudahLogin ? 'Selamat datang kembali!' : 'Silakan login.'}</p>
    <p>Hasil: {2 + 2}</p>
  </div>
);
  • Tag Harus Ditutup (Self-Closing Tags): Semua tag JSX harus ditutup. Untuk elemen yang tidak memiliki konten anak (elemen kosong seperti <img />, <br />, <input />), Anda harus menggunakan format self-closing tag dengan menambahkan /> di akhir.
  • Komentar di JSX: Komentar di dalam JSX ditulis sebagai ekspresi JavaScript yang dibungkus kurung kurawal dan blok komentar JavaScript: {/* Ini adalah komentar di dalam JSX */}.

üé® Styling di React

React tidak memaksakan satu cara tertentu untuk menata gaya komponen Anda, memberikan fleksibilitas untuk memilih metode yang paling sesuai dengan proyek dan preferensi tim Anda. Mari kita lihat beberapa pendekatan populer.

1. CSS Biasa (Plain CSS)

Ini adalah pendekatan paling tradisional. Anda membuat file .css terpisah (misalnya, MyComponent.css) dan mengimpornya ke dalam file komponen JavaScript/JSX Anda. Nama kelas CSS kemudian diterapkan menggunakan atribut className di JSX.

// MyComponent.jsx
import './MyComponent.css'; // Impor file CSS

function MyComponent() {
  return <div className="my-component-style">Konten Komponen Saya</div>;
}
export default MyComponent;
/* MyComponent.css */
.my-component-style {
  background-color: #4ECDC4; /* Turquoise Accent */
  color: white;
  padding: 1rem;
  border-radius: 0.5rem;
  text-align: center;
}

Kelebihan: Sangat familiar bagi developer yang terbiasa dengan CSS tradisional, mudah dimulai.
Kekurangan: Secara default, semua style bersifat global. Ini dapat menyebabkan konflik nama kelas jika tidak dikelola dengan hati-hati, terutama pada aplikasi yang besar dan kompleks.

2. CSS Modules

CSS Modules adalah file CSS di mana semua nama kelas dan animasi memiliki scope lokal secara default. Ini berarti style yang Anda tulis untuk satu komponen tidak akan secara tidak sengaja memengaruhi komponen lain. Nama file CSS biasanya diakhiri dengan .module.css (misalnya, MyButton.module.css).

// MyButton.jsx
import styles from './MyButton.module.css'; // Impor CSS Module

function MyButton() {
  return <button className={styles.buttonError}>Tombol Error</button>;
}
export default MyButton;
/* MyButton.module.css */
.buttonError { /* Nama kelas ini akan diubah menjadi unik, misal: MyButton_buttonError__a1b2c */
  background-color: #FF6B6B; /* Coral Primary */
  color: white;
  padding: 0.75rem 1.5rem;
  border: none;
  border-radius: 0.375rem;
}

Kelebihan: Mencegah konflik nama kelas global karena nama kelas diubah menjadi unik secara otomatis. Menjaga style tetap terenkapsulasi per komponen.
Kekurangan: Penamaan kelas di JSX menjadi sedikit lebih panjang ({styles.namaKelas}).

3. Inline Styles

Style dapat diterapkan langsung ke elemen JSX menggunakan atribut style. Atribut ini menerima objek JavaScript, di mana properti CSS ditulis dalam format camelCase (misalnya, backgroundColor bukan background-color).

function HighlightedText({ text, isActive }) {
  const dynamicStyle = {
    backgroundColor: isActive ? '#4ECDC4' : '#F7F7F7', // Turquoise atau Light Gray
    color: isActive ? 'white' : '#4A4A4A',
    padding: '10px',
    borderRadius: '5px', // camelCase untuk border-radius
    fontSize: '1.1rem'   // camelCase untuk font-size
  };

  return <p style={dynamicStyle}>{text}</p>;
}
export default HighlightedText;

Kelebihan: Sangat berguna untuk style yang sangat spesifik atau dinamis yang bergantung pada state atau props komponen. Scope style hanya pada elemen tersebut.
Kekurangan: Kurang ideal untuk style yang kompleks. Tidak mendukung semua fitur CSS (seperti pseudo-classes seperti :hover atau media queries secara langsung tanpa trik tambahan). Bisa membuat JSX kurang terbaca jika terlalu banyak style inline.

4. CSS-in-JS Libraries (Contoh: Styled Components)

Pustaka CSS-in-JS seperti Styled Components atau Emotion memungkinkan Anda menulis CSS aktual langsung di dalam file JavaScript/JSX Anda, seringkali menggunakan *template literals*. Komponen yang Anda buat sudah memiliki style-nya sendiri secara terenkapsulasi.

Untuk menggunakan Styled Components, Anda perlu menginstalnya terlebih dahulu: npm install styled-components atau yarn add styled-components.

// Button.jsx (menggunakan Styled Components)
import styled from 'styled-components';

const TombolKustom = styled.button`
  background-color: ${props => props.primary ? '#FF6B6B' : '#4ECDC4'}; /* Coral atau Turquoise */
  color: white;
  font-size: 1em;
  margin: 1em;
  padding: 0.75em 1.5em;
  border: none;
  border-radius: 0.375rem;
  cursor: pointer;
  transition: all 0.3s ease;

  &:hover {
    opacity: 0.85;
    transform: translateY(-2px);
  }
`;

// Penggunaan di komponen lain:
// import TombolKustom from './Button';
// ...
// <TombolKustom primary>Tombol Utama</TombolKustom>
// <TombolKustom>Tombol Sekunder</TombolKustom>

Kelebihan: Enkapsulasi style yang sangat kuat (style ter-scope ke komponen). Style dinamis berdasarkan props menjadi sangat mudah. Fitur seperti theming sering tersedia. Menghilangkan kebutuhan untuk memetakan antara komponen dan style secara manual.
Kekurangan: Menambah dependensi pada proyek dan sedikit menambah ukuran bundle aplikasi. Bisa ada sedikit overhead performa (meskipun seringkali dapat diabaikan untuk kebanyakan aplikasi). Membutuhkan kurva belajar tambahan untuk memahami API pustaka tersebut.

Pilihan lain yang sangat populer dan direkomendasikan adalah menggunakan framework CSS utility-first seperti Tailwind CSS (yang sedang kita gunakan di panduan ini!). Tailwind memungkinkan Anda membangun desain kustom dengan cepat tanpa meninggalkan HTML Anda.

üîó Interaksi Komponen: State & Props

State dan Props adalah dua konsep fundamental di React yang memungkinkan komponen untuk berinteraksi, mengelola data internalnya, dan menerima data dari luar, sehingga membuat antarmuka pengguna menjadi dinamis dan responsif.

Props (Properties) üéÅ

Props (kependekan dari *properties*) adalah cara komponen menerima data dari komponen induknya (parent component). Bayangkan props sebagai argumen yang Anda teruskan ke sebuah fungsi. Props bersifat read-only di dalam komponen yang menerimanya; artinya, komponen anak tidak boleh memodifikasi props yang diterimanya secara langsung. Jika data perlu diubah, perubahan tersebut harus berasal dari komponen yang memiliki data tersebut (biasanya komponen induk).

Contoh Penggunaan Props:

// Komponen Induk (misalnya, App.jsx)
function App() {
  return <UserProfile nama="Andi Pratama" usia={28} pekerjaan="Developer" hobi={['Membaca', 'Koding']} />;
}

// Komponen Anak (UserProfile.jsx)
// Menerima props sebagai argumen objek tunggal
function UserProfile(props) {
  // props adalah objek: { nama: "Andi Pratama", usia: 28, pekerjaan: "Developer", hobi: ['Membaca', 'Koding'] }
  return (
    <div className="profile-card p-4 border rounded-md bg-gray-50">
      <h4 className="text-xl font-semibold text-gray-800 mb-2">Profil Pengguna</h4>
      <p className="text-gray-700">Nama: {props.nama}</p>
      <p className="text-gray-700">Usia: {props.usia} tahun</p>
      <p className="text-gray-700">Pekerjaan: {props.pekerjaan}</p>
      <p className="text-gray-700">Hobi: {props.hobi.join(', ')}</p>
    </div>
  );
}

// Alternatif yang lebih umum: Destructuring props langsung di parameter fungsi
// function UserProfile({ nama, usia, pekerjaan, hobi }) {
//   return ( /* ... JSX yang sama menggunakan variabel nama, usia, dst. secara langsung ... */ );
// }
export default App; // (Contoh export, sesuaikan dengan struktur file Anda)

State ‚ù§Ô∏è

State adalah data yang dimiliki dan dikelola *secara internal* oleh sebuah komponen. Berbeda dengan props, state bersifat **mutable** (bisa diubah) oleh komponen itu sendiri, biasanya sebagai respons terhadap interaksi pengguna (seperti klik tombol atau input form) atau kejadian lain dalam aplikasi (seperti data yang diterima dari API).

Ketika state sebuah komponen berubah, React akan secara otomatis **me-render ulang** (re-render) komponen tersebut (dan anak-anaknya jika perlu) untuk mencerminkan perubahan tersebut di antarmuka pengguna.

Menggunakan Hook `useState`:

Di *functional components* (cara modern dan paling umum untuk menulis komponen React), state dikelola menggunakan *Hook* useState. Hook useState adalah fungsi yang:

  1. Menerima satu argumen: nilai awal (initial state) dari state tersebut.
  2. Mengembalikan sebuah array dengan dua elemen:
    • Elemen pertama: Nilai state saat ini.
    • Elemen kedua: Fungsi untuk memperbarui nilai state tersebut. (Konvensi penamaannya adalah setNamaState).

import React, { useState } from 'react'; // Jangan lupa import useState

function Counter() {
  // Deklarasi state 'hitungan' dengan nilai awal 0
  // 'hitungan' adalah nilai state saat ini
  // 'setHitungan' adalah fungsi untuk memperbarui 'hitungan'
  const [hitungan, setHitungan] = useState(0);

  const tambahHitungan = () => {
    setHitungan(prevHitungan => prevHitungan + 1); // Cara aman update berdasarkan state sebelumnya
  };

  const resetHitungan = () => {
    setHitungan(0);
  };

  return (
    <div className="p-4 border rounded-md bg-gray-50">
      <p className="text-xl mb-3 text-gray-700">Anda telah mengklik tombol sebanyak: <span className="font-bold" style={{ color: '#FF6B6B' }}>{hitungan}</span> kali</p>
      <button 
        className="bg-emerald-500 hover:bg-emerald-600 text-white font-bold py-2 px-4 rounded mr-2 transition-colors"
        onClick={tambahHitungan}
      >
        Tambah (+)
      </button>
      <button 
        className="bg-gray-400 hover:bg-gray-500 text-white font-bold py-2 px-4 rounded transition-colors"
        onClick={resetHitungan}
      >
        Reset
      </button>
    </div>
  );
}
export default Counter;

Props vs. State - Kapan Menggunakan yang Mana? ü§î

Memahami perbedaan dan kapan menggunakan props atau state adalah kunci untuk membangun aplikasi React yang terstruktur dengan baik.

Fitur Props State
Sumber Data Diterima dari komponen induk (data eksternal bagi komponen penerima). Dikelola di dalam komponen itu sendiri (data internal).
Kemampuan Perubahan Read-only (tidak bisa diubah secara langsung oleh komponen anak yang menerimanya). Mutable (bisa diubah oleh komponen itu sendiri menggunakan fungsi set-nya, misal, setHitungan).
Tujuan Utama Untuk mengonfigurasi komponen dan meneruskan data atau fungsi dari komponen induk ke komponen anak. Untuk mengelola data yang berubah seiring waktu akibat interaksi pengguna, respons API, atau logika internal komponen, yang menyebabkan UI diperbarui.
Siapa yang "Memiliki"? Komponen anak "menerima" dari komponen induk. Komponen induk "memiliki" data yang diteruskan sebagai props. Komponen "memiliki" dan "mengontrol" datanya sendiri.
Contoh Penggunaan <UserProfile nama="Budi" /> const [isOpen, setIsOpen] = useState(false);

"Lifting State Up" (Mengangkat State ke Atas): Jika beberapa komponen anak perlu berbagi atau memodifikasi state yang sama, praktik terbaiknya adalah "mengangkat" state tersebut ke komponen induk terdekat mereka. Komponen induk kemudian akan meneruskan state tersebut (sebagai props) dan fungsi untuk mengubah state tersebut (juga sebagai props) ke komponen-komponen anak yang membutuhkannya.

üìù Form & List: Mengelola Input dan Menampilkan Data

Mengelola input pengguna melalui formulir dan menampilkan daftar data secara dinamis adalah dua tugas yang sangat umum dalam pengembangan aplikasi web. React menyediakan cara yang elegan dan efektif untuk menangani keduanya.

Mengelola Formulir dengan "Controlled Components"

Di React, pendekatan yang paling umum dan direkomendasikan untuk menangani formulir adalah dengan menggunakan konsep **"Controlled Components"**. Dalam *controlled component*, data formulir (nilai dari setiap input field) dikelola oleh state React, bukan oleh DOM itu sendiri.

  • Setiap elemen input (seperti <input>, <textarea>, <select>) memiliki atribut value yang diikat (bound) ke sebuah variabel state di komponen React.
  • Setiap kali nilai input berubah (misalnya, pengguna mengetik sesuatu), fungsi *event handler* (biasanya yang terikat pada atribut onChange) akan dipicu.
  • Fungsi onChange ini akan memperbarui state React dengan nilai baru dari input.
  • Karena state berubah, komponen akan di-render ulang, dan elemen input akan menampilkan nilai baru dari state tersebut. Ini memastikan bahwa state React selalu menjadi "sumber kebenaran" (source of truth) untuk nilai form.

Contoh Kode Formulir Sederhana (Controlled Component):

import React, { useState } from 'react';

function NameForm() {
  const [nama, setNama] = useState(''); // State untuk nilai input

  const handleChange = (event) => {
    setNama(event.target.value); // Update state saat input berubah
  };

  const handleSubmit = (event) => {
    event.preventDefault(); // Mencegah reload halaman standar saat form disubmit
    if (nama.trim() === '') {
        alert('Nama tidak boleh kosong!');
        return;
    }
    alert('Nama yang dikirim: ' + nama);
    setNama(''); // Kosongkan input setelah submit
  };

  return (
    <form onSubmit={handleSubmit} className="space-y-4 p-4 border border-gray-300 rounded-lg bg-gray-50">
      <div>
        <label htmlFor="nameInput" className="block text-sm font-medium text-gray-700 mb-1">
          Masukkan Nama Anda:
        </label>
        <input
          type="text"
          id="nameInput"
          value={nama} // Nilai input dikontrol oleh state
          onChange={handleChange} // Fungsi untuk update state saat ada perubahan
          className="mt-1 block w-full px-3 py-2 bg-white border border-gray-300 rounded-md shadow-sm focus:outline-none focus:ring-coral-500 focus:border-coral-500 sm:text-sm"
          placeholder="Contoh: Budi Doremi"
        />
      </div>
      <button 
        type="submit"
        className="bg-coral-500 hover:bg-coral-600 text-white font-bold py-2 px-4 rounded transition-colors"
      >
        Kirim
      </button>
    </form>
  );
}
export default NameForm;

Menampilkan Daftar (Lists) Data

Untuk me-render daftar (list) elemen di React, Anda biasanya akan memiliki sebuah array data dan menggunakan metode .map() dari array JavaScript. Metode .map() akan mengiterasi setiap item dalam array dan memungkinkan Anda mengembalikan elemen JSX untuk setiap item tersebut.

PENTING: Atribut key!

Setiap kali Anda me-render daftar elemen menggunakan .map(), React memerlukan atribut key yang **unik dan stabil** untuk setiap elemen dalam daftar tersebut. key membantu React mengidentifikasi item mana yang telah berubah, ditambahkan, atau dihapus. Ini sangat penting untuk performa dan untuk menjaga state internal dari item list (jika ada) tetap konsisten.

Biasanya, ID unik dari data Anda (misalnya, ID dari database atau UUID yang digenerate) digunakan sebagai key. **Hindari menggunakan index array sebagai key jika urutan item dalam daftar dapat berubah**, karena ini dapat menyebabkan masalah performa dan state yang tidak konsisten.

Contoh Kode Menampilkan Daftar:

function ShoppingList({ items }) {
  // items adalah array objek, misal:
  // const daftarBelanja = [
  //   { id: 'item-001', name: 'Apel Merah', quantity: 5 },
  //   { id: 'item-002', name: 'Susu UHT Full Cream', quantity: 2 },
  //   { id: 'item-003', name: 'Roti Tawar Gandum', quantity: 1 }
  // ];

  if (!items || items.length === 0) {
    return <p className="text-gray-500 italic">Tidak ada barang di daftar belanja Anda saat ini.</p>;
  }

  return (
    <div className="p-4 border rounded-md bg-gray-50">
      <h4 className="text-xl font-semibold mb-3 text-gray-700">Daftar Belanja Anda:</h4>
      <ul className="list-disc list-inside space-y-2">
        {items.map(item => (
          // 'item.id' digunakan sebagai key yang unik dan stabil
          <li key={item.id} className="py-1 text-gray-800">
            {item.name} <span className="text-sm text-gray-600">(Jumlah: {item.quantity})</span>
          </li>
        ))}
      </ul>
    </div>
  );
}
export default ShoppingList;

üî® Studi Kasus: Aplikasi To-Do List

Saatnya menerapkan semua konsep yang telah kita pelajari untuk membangun sebuah aplikasi interaktif sederhana: Aplikasi To-Do List. Ini adalah latihan yang sangat baik untuk mempraktikkan penggunaan komponen, state, props, formulir, dan rendering daftar.

Fitur Utama Aplikasi To-Do List

Aplikasi To-Do List yang akan kita rancang akan memiliki fungsionalitas dasar sebagai berikut:

  • Menambah Task Baru: Pengguna dapat mengetikkan deskripsi task baru di sebuah input field dan menambahkannya ke dalam daftar task.
  • Menampilkan Daftar Tasks: Semua task yang telah ditambahkan akan ditampilkan dalam sebuah daftar yang terstruktur.
  • Menandai Task Selesai/Belum Selesai (Toggle): Setiap task dalam daftar dapat ditandai sebagai sudah selesai (misalnya dengan mencoret teksnya atau mengubah warnanya) atau belum selesai. Pengguna dapat mengubah status ini.
  • Menghapus Task: Pengguna dapat menghapus task tertentu dari daftar.

Struktur Komponen yang Direncanakan (Saran)

Untuk membangun aplikasi To-Do List ini dengan baik, kita bisa memecahnya menjadi beberapa komponen yang lebih kecil dan terkelola:

  • App.jsx: Ini akan menjadi komponen induk (root component) yang mengatur keseluruhan aplikasi. Komponen ini biasanya akan bertanggung jawab untuk menampung state utama (yaitu, array dari semua tasks) dan logika utama untuk menambah, menghapus, serta mengubah status (selesai/belum) dari sebuah task.
  • TodoForm.jsx: Komponen ini akan berisi formulir (input field dan tombol tambah) untuk pengguna memasukkan task baru. Komponen ini akan menerima fungsi dari App (melalui props) untuk benar-benar menambahkan task baru ke state utama.
  • TodoList.jsx: Komponen ini bertanggung jawab untuk menampilkan daftar semua tasks. Ia akan menerima daftar tasks (dari state di App) dan fungsi-fungsi terkait (seperti fungsi untuk menghapus atau mengubah status task) melalui props. Kemudian, komponen ini akan me-render setiap task menggunakan komponen TodoItem.
  • TodoItem.jsx: Komponen ini merepresentasikan satu task individual dalam daftar. Ia akan menerima detail dari satu task (seperti teks dan status selesai) dan fungsi-fungsi terkait (untuk menghapus atau mengubah status) dari TodoList (atau bisa juga langsung dari App) melalui props. Komponen ini juga akan berisi UI untuk menampilkan teks task, checkbox (atau tombol) untuk menandai selesai, dan tombol untuk menghapus.

Struktur Data Task (Contoh)

Setiap task dalam daftar To-Do List kita bisa direpresentasikan sebagai sebuah objek JavaScript. Objek ini setidaknya harus memiliki properti berikut:

  • id: Sebuah identifier unik (bisa berupa string atau number) untuk setiap task. Ini sangat penting untuk digunakan sebagai atribut key saat me-render daftar task, dan juga untuk mengidentifikasi task mana yang akan dihapus atau diubah statusnya.
  • text: Sebuah string yang berisi deskripsi dari task yang akan ditampilkan kepada pengguna.
  • isCompleted: Sebuah boolean yang menandakan apakah task tersebut sudah selesai (true) atau belum selesai (false). Nilai defaultnya biasanya false saat task baru dibuat.

  { id: 'todo-3', text: 'Mengerjakan tugas styling', isCompleted: false }

Poin-Poin Kunci yang Telah Dipelajari

Mari kita tinjau kembali beberapa poin kunci yang telah Anda pelajari dalam panduan ini:

  • React adalah Pustaka UI Berbasis Komponen: Fokus utama React adalah membangun antarmuka pengguna dari komponen-komponen yang independen dan dapat digunakan kembali, dengan pendekatan deklaratif.
  • Vite Mempercepat Pengembangan: Vite adalah alat bantu modern yang menyediakan server pengembangan super cepat dan Hot Module Replacement (HMR) instan, meningkatkan produktivitas developer React.
  • JSX Memudahkan Penulisan UI: JSX adalah ekstensi sintaks yang memungkinkan Anda menulis markup mirip HTML di dalam JavaScript, membuat kode UI lebih intuitif dan mudah dibaca.
  • Fleksibilitas dalam Styling: React tidak memaksakan satu cara styling; Anda bisa memilih dari CSS biasa, CSS Modules, inline styles, hingga pustaka CSS-in-JS atau framework utility-first seperti Tailwind CSS.
  • State & Props adalah Inti Dinamisme: State digunakan untuk mengelola data internal komponen yang bisa berubah, sementara Props digunakan untuk meneruskan data (dan fungsi) dari komponen induk ke anak secara read-only.
  • Pola Umum Form & List: Mengelola formulir dengan "Controlled Components" (di mana state React menjadi sumber kebenaran) dan menampilkan daftar data menggunakan metode .map() dengan atribut key yang unik adalah pola yang sangat umum dan penting dalam pengembangan React.

Langkah Selanjutnya dalam Perjalanan React Anda üöÄ

Perjalanan belajar React Anda baru saja dimulai, dan ada banyak hal menarik lainnya yang bisa Anda eksplorasi untuk memperdalam pemahaman dan keahlian Anda:

  1. Praktik, Praktik, dan Lebih Banyak Praktik: Cara terbaik untuk benar-benar menguasai React adalah dengan membangun proyek. Mulailah dengan mengimplementasikan aplikasi To-Do List yang telah kita diskusikan secara penuh. Coba tambahkan fitur-fitur baru seperti filter tasks (semua, aktif, selesai) atau kemampuan untuk mengedit task yang sudah ada.
  2. Pelajari Hooks React Lebih Lanjut: Selain useState, ada banyak Hooks lain yang sangat berguna:
    • useEffect: Untuk menangani *side effects* dalam komponen fungsional, seperti pengambilan data dari API (data fetching), manipulasi DOM secara manual (meskipun sebaiknya dihindari), atau berlangganan (subscribe) dan berhenti berlangganan (unsubscribe) ke event.
    • useContext: Untuk berbagi state antar komponen tanpa perlu meneruskannya secara manual melalui props di setiap level hirarki komponen (membantu menghindari *prop drilling*).
    • Hooks lainnya seperti useReducer (untuk manajemen state yang lebih kompleks), useCallback dan useMemo (untuk optimasi performa), serta useRef (untuk mengakses elemen DOM secara langsung atau menyimpan nilai mutable yang tidak memicu re-render).
  3. Pelajari tentang Routing di React: Untuk membangun aplikasi multi-halaman (Single Page Applications dengan banyak "view" atau "halaman" virtual), Anda akan memerlukan pustaka routing. Pilihan paling populer adalah React Router.
  4. Coba Pustaka State Management yang Lebih Canggih: Jika aplikasi Anda tumbuh semakin kompleks dan manajemen state menjadi rumit hanya dengan useState dan useContext, pertimbangkan untuk mempelajari pustaka state management global seperti Zustand, Jotai, atau Redux (meskipun Redux memiliki kurva belajar yang lebih curam).
  5. Eksplorasi Ekosistem React: Coba gunakan pustaka komponen UI seperti Material-UI, Ant Design, atau Chakra UI untuk mempercepat pengembangan UI. Pelajari juga tentang testing komponen React menggunakan pustaka seperti Jest dan React Testing Library.
  6. Berkontribusi dan Bergabung dengan Komunitas: Cara hebat untuk belajar adalah dengan terlibat dalam komunitas React. Ikuti blog, forum diskusi (seperti Reddit r/reactjs), tonton konferensi, dan jika Anda sudah merasa cukup percaya diri, cobalah berkontribusi pada proyek open-source.
