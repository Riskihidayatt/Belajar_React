üåê Pengenalan React & Vite

Bagian ini akan memandu Anda memahami apa itu React, mengapa ia begitu populer, dan bagaimana cara cepat memulai proyek menggunakan Vite.

Apa itu React? ‚öõÔ∏è

React adalah sebuah pustaka JavaScript yang deklaratif, efisien, dan fleksibel untuk membangun antarmuka pengguna (UI) yang kompleks dan interaktif. Dikembangkan oleh Facebook, React memungkinkan developer untuk membuat komponen UI yang dapat digunakan kembali (reusable components).

Pendekatan deklaratif React berarti Anda cukup mendeskripsikan bagaimana UI seharusnya terlihat berdasarkan data (state) saat ini, dan React akan secara otomatis memperbarui DOM ketika data tersebut berubah.

Mengapa React Begitu Populer?

  • Virtual DOM: React menggunakan Virtual DOM untuk optimasi performa. Perubahan pada UI pertama kali diterapkan pada representasi DOM di memori (Virtual DOM), kemudian React menghitung cara paling efisien untuk memperbarui DOM browser yang sebenarnya. Ini meminimalkan manipulasi DOM langsung yang mahal.
  • Berbasis Komponen: Memecah UI menjadi komponen-komponen kecil yang independen dan dapat digunakan kembali membuat kode lebih modular, mudah dikelola, dan diskalakan.
  • Ekosistem Besar: Komunitas yang besar dan aktif berarti banyak sumber daya, pustaka pihak ketiga (seperti React Router untuk navigasi, Redux/Zustand untuk state management), dan dukungan yang tersedia.
  • Unidirectional Data Flow: Aliran data satu arah (biasanya dari komponen induk ke anak melalui props) membuat state aplikasi lebih mudah diprediksi dan di-debug.

Memulai dengan Vite ‚ö°

Vite adalah alat bantu pengembangan web modern yang memberikan pengalaman pengembangan frontend yang sangat cepat. Ia menyediakan server pengembangan dengan Hot Module Replacement (HMR) instan dan build yang dioptimalkan.

Untuk membuat proyek React baru dengan Vite:

npm create vite@latest nama-proyek-react -- --template react

Setelah proyek dibuat, navigasikan ke direktori proyek dan instal dependensi: 

cd nama-proyek-react
npm install

Kemudian, jalankan server pengembangan:

npm run dev

Vite akan memulai server dan aplikasi React Anda akan dapat diakses di browser, biasanya di http://localhost:5173.

‚ú® JSX: JavaScript XML

JSX adalah ekstensi sintaks untuk JavaScript yang memungkinkan Anda menulis markup mirip HTML langsung di dalam file JavaScript Anda. Ini membuat pembuatan komponen React lebih intuitif.

Memahami JSX

Meskipun terlihat seperti HTML, JSX sebenarnya adalah "syntactic sugar" untuk fungsi `React.createElement(component, props, ...children)`. Browser tidak memahami JSX secara langsung; ia perlu dikompilasi menjadi JavaScript biasa oleh transpiler seperti Babel (yang sudah terintegrasi dalam Vite).

Contoh JSX:

const nama = "Pengguna React";
const elemen = <h1 className="salam">Halo, {nama}!</h1>;

// JSX di atas akan dikompilasi menjadi:
// React.createElement("h1", {className: "salam"}, "Halo, ", nama, "!");

Aturan Penting dalam JSX

  • Satu Elemen Induk (Root): Komponen React harus mengembalikan satu elemen induk. Jika Anda memiliki beberapa elemen, bungkus dalam satu `<div>` atau gunakan `<React.Fragment>` (atau sintaks singkatnya `<>...</>`).
  • Atribut HTML: Banyak atribut HTML standar berfungsi di JSX, tetapi beberapa ditulis dalam camelCase (misalnya, `onClick`, `onSubmit`). Atribut `class` ditulis sebagai `className` dan `for` sebagai `htmlFor`.
  • Ekspresi JavaScript: Anda dapat menyematkan ekspresi JavaScript apa pun di dalam JSX dengan menggunakan kurung kurawal `{ }`. Misalnya, `{user.name}`, `{2 + 2}`, `{isAdmin ? : }`.
  • Self-Closing Tags: Elemen yang tidak memiliki konten anak (seperti `<img />`, `<br />`, `<input />`) harus ditutup dengan ` />`.
  • Komentar: Komentar di JSX ditulis sebagai `{/* Ini komentar */}`.

üé® Styling di React

Ada berbagai cara untuk menata gaya komponen React Anda. Bagian ini akan membahas beberapa metode populer.

1. CSS Biasa (Plain CSS)

Anda dapat membuat file `.css` dan mengimpornya ke komponen Anda. Nama kelas diterapkan menggunakan atribut `className`. 

// MyComponent.jsx
import './MyComponent.css';

function MyComponent() {
  return <div className="my-component-style">Konten Komponen</div>;
}
/* MyComponent.css */
.my-component-style {
  background-color: #2D3748; /* gray-800 versi lain */
  color: #E2E8F0; /* slate-200 */
  padding: 1rem;
  border-radius: 0.5rem;
}

Kelemahan: Scope CSS bersifat global, yang dapat menyebabkan konflik nama kelas pada aplikasi besar.

2. CSS Modules

CSS Modules adalah file CSS di mana semua nama kelas dan animasi memiliki scope lokal secara default (nama file biasanya `*.module.css`). 

// MyComponent.jsx
import styles from './MyComponent.module.css';

function MyComponent() {
  return <div className={styles.myComponentStyle}>Konten dengan CSS Module</div>;
}
/* MyComponent.module.css */
.myComponentStyle { /* Akan menjadi unik seperti MyComponent_myComponentStyle__XYZ123 */
  background-color: #4A5568; /* gray-700 versi lain */
  color: #F7FAFC; /* gray-100 */
  padding: 1rem;
  border: 1px solid #718096; /* gray-500 */
}

Kelebihan: Mencegah konflik nama kelas global.

3. Inline Styles

Style diterapkan langsung ke elemen JSX menggunakan atribut `style`, yang menerima objek JavaScript dengan properti camelCase. 

function MyComponent({ isActive }) {
  const inlineStyle = {
    backgroundColor: isActive ? '#38A169' : '#4A5568', // emerald-600 atau gray-700
    color: 'white',
    padding: '10px 20px',
    borderRadius: '5px',
    fontSize: '1rem' 
  };
  return <button style={inlineStyle}>Tombol Dinamis</button>;
}

Kelebihan: Berguna untuk style dinamis. Kekurangan: Kurang ideal untuk style kompleks, tidak mendukung pseudo-class secara langsung.

4. CSS-in-JS (Contoh: Styled Components)

Pustaka seperti Styled Components memungkinkan Anda menulis CSS aktual di dalam file JavaScript menggunakan template literals.

Pertama, instal: npm install styled-components 

// Button.jsx
import styled from 'styled-components';

const StyledButton = styled.button`
  background-color: ${props => props.primary ? '#06B6D4' : '#475569'}; /* sky-500 atau slate-600 */
  color: white;
  font-size: 1em;
  padding: 0.5em 1.5em;
  border: none;
  border-radius: 0.375rem;
  cursor: pointer;
  transition: background-color 0.3s ease;

  &:hover {
    background-color: ${props => props.primary ? '#0891B2' : '#334155'}; /* sky-600 atau slate-700 */
  }
`;

// Penggunaan:
// <StyledButton primary>Tombol Utama</StyledButton>
// <StyledButton>Tombol Sekunder</StyledButton>

Kelebihan: Enkapsulasi style yang kuat, style dinamis berdasarkan props. Kekurangan: Menambah dependensi, kurva belajar tambahan.

üîó Interaksi Komponen: State & Props

State dan Props adalah jantung dari aplikasi React yang dinamis, memungkinkan komponen untuk mengelola data dan berkomunikasi.

Props (Properties) üéÅ

Props adalah cara komponen menerima data dari komponen induknya. Props bersifat read-only di dalam komponen yang menerimanya; komponen anak tidak boleh memodifikasi props secara langsung. 

// Komponen Induk
function App() {
  return <SalamPengguna nama="Budi" usia={30} />;
}

// Komponen Anak
function SalamPengguna(props) {
  // props adalah objek: { nama: "Budi", usia: 30 }
  return (
    <div>
      <p>Halo, {props.nama}! Usia Anda {props.usia} tahun.</p>
    </div>
  );
}

// Bisa juga dengan destructuring props:
// function SalamPengguna({ nama, usia }) { ... }

State ‚ù§Ô∏è

State adalah data internal yang dimiliki dan dikelola oleh sebuah komponen. Ketika state berubah, React akan me-render ulang komponen tersebut untuk mencerminkan perubahan UI.

Di functional components, state dikelola menggunakan Hook `useState`. 

import React, { useState } from 'react';

function Counter() {
  // Deklarasi state 'hitungan' dengan nilai awal 0
  // hitungan: nilai state saat ini
  // setHitungan: fungsi untuk memperbarui state 'hitungan'
  const [hitungan, setHitungan] = useState(0);

  return (
    <div>
      <p className="text-lg mb-2">Nilai hitungan: {hitungan}</p>
      <button 
        onClick={() => setHitungan(hitungan + 1)}
        className="bg-emerald-500 hover:bg-emerald-600 text-white font-semibold py-2 px-4 rounded mr-2"
      >
        Tambah
      </button>
      <button 
        onClick={() => setHitungan(prevHitungan => prevHitungan - 1)} // Cara aman update berdasarkan state sebelumnya
        className="bg-sky-500 hover:bg-sky-600 text-white font-semibold py-2 px-4 rounded"
      >
        Kurang
      </button>
    </div>
  );
}

Props vs. State ü§î

Fitur Props State
Sumber Data Eksternal (dari komponen induk) Internal (dikelola komponen sendiri)
Sifat Read-only (tidak bisa diubah anak) Mutable (bisa diubah oleh komponen)
Tujuan Meneruskan data & konfigurasi ke anak Mengelola data dinamis internal

Lifting State Up: Jika beberapa komponen perlu berbagi state, state tersebut "diangkat" ke komponen induk terdekat dan diteruskan ke anak melalui props.

üìù Form & List

Mengelola input pengguna melalui form dan menampilkan daftar data adalah tugas umum dalam aplikasi web.

Formulir (Controlled Components)

Dalam React, pendekatan umum untuk form adalah menggunakan "Controlled Components", di mana data form dikelola oleh state React. 

import React, { useState } from 'react';

function MyForm() {
  const [inputValue, setInputValue] = useState('');

  const handleChange = (event) => {
    setInputValue(event.target.value);
  };

  const handleSubmit = (event) => {
    event.preventDefault(); // Mencegah reload halaman
    alert('Nilai yang disubmit: ' + inputValue);
    setInputValue(''); // Reset input
  };

  return (
    <form onSubmit={handleSubmit} className="space-y-4">
      <label htmlFor="myInput" className="block text-lg font-medium">Masukkan Teks:</label>
      <input
        type="text"
        id="myInput"
        value={inputValue} // Nilai input dikontrol oleh state
        onChange={handleChange} // Fungsi untuk update state
        className="block w-full p-3 bg-slate-700 border border-slate-600 rounded-md focus:ring-sky-500 focus:border-sky-500"
      />
      <button type="submit" className="bg-emerald-500 hover:bg-emerald-600 text-white font-semibold py-2 px-6 rounded-md">
        Kirim
      </button>
    </form>
  );
}

Menampilkan Daftar (Lists)

Untuk me-render daftar elemen, gunakan metode `.map()` pada array. Penting untuk memberikan atribut `key` yang unik dan stabil untuk setiap item list agar React dapat mengelola pembaruan secara efisien. 

function ItemList({ items }) {
  // items adalah array objek, contoh: [{ id: 'a1', name: 'Apel' }, { id: 'b2', name: 'Jeruk' }]
  if (!items || items.length === 0) {
    return <p>Tidak ada item untuk ditampilkan.</p>;
  }
  return (
    <ul className="list-disc list-inside space-y-2 p-4 bg-slate-700 rounded-md">
      {items.map((item) => (
        <li key={item.id} className="text-slate-300"> 
          {item.name} 
        </li>
        // Key 'item.id' harus unik dan stabil
      ))}
    </ul>
  );
}

// Contoh penggunaan:
// const daftarBuah = [{ id: 'f1', name: 'Mangga' }, { id: 'f2', name: 'Pisang' }];
// <ItemList items={daftarBuah} />

Hindari menggunakan index array sebagai `key` jika urutan item dapat berubah, karena dapat menyebabkan masalah performa dan state yang tidak konsisten.

Fitur Utama

üèÅ Kesimpulan & Langkah Selanjutnya

Selamat! Anda telah menjelajahi konsep-konsep inti React. Ini adalah fondasi yang kuat untuk membangun aplikasi web modern.

Poin Kunci yang Dipelajari

  • React adalah pustaka UI berbasis komponen yang deklaratif.
  • Vite mempercepat development React.
  • JSX memudahkan penulisan UI.
  • Berbagai metode styling dapat digunakan sesuai kebutuhan.
  • State & Props adalah inti dari data dinamis dan interaksi komponen.
  • Mengelola form (controlled components) dan list (dengan `.map()` dan `key`) adalah pola umum.

Langkah Selanjutnya üöÄ

  1. Praktik Lebih Banyak: Implementasikan aplikasi To-Do List secara penuh. Coba tambahkan fitur seperti filter atau edit task.
  2. Pelajari Hooks Lebih Lanjut: Eksplorasi `useEffect` (untuk side effects seperti data fetching), `useContext` (untuk state management sederhana tanpa prop drilling), `useReducer`, `useCallback`, `useMemo`.
  3. Routing: Pelajari React Router untuk membangun aplikasi multi-halaman (SPA dengan banyak view).
  4. State Management Lanjutan: Untuk aplikasi kompleks, pelajari pustaka state management global seperti Zustand, Jotai, atau Redux.
  5. Eksplorasi Ekosistem: Coba pustaka UI seperti Material-UI atau Chakra UI. Pelajari tentang testing komponen React.
  6. Kontribusi & Komunitas: Bergabung dengan komunitas React, baca blog, dan jika memungkinkan, berkontribusi pada proyek open-source.

Teruslah belajar dan membangun. Semoga sukses dalam perjalanan React Anda!
